Browser-First VFR Planning

Plan cross-country flights with weather, checkpoints, route maps, airspace, and airport diagrams in one workflow.

CieloRumbo is an educational VFR planning tool for pilots and students. It combines airport weather, aircraft performance, dead-reckoning checkpoints, route review, FAA airspace overlays, airport diagrams, and altitude profiling into one browser-based workflow.

Features

What CieloRumbo already covers

Flight Setup

Build multi-leg routes, select aircraft profiles, pick planned altitudes, and generate a navigation log from a browser workflow.

Weather-Aware Planning

Pull FAA weather and forecast data, populate route airports, and surface route-relevant weather directly in planning and map views.

Checkpoint Planning

Generate and refine route checkpoints, review them visually, and push approved checkpoints back into the nav log.

Route Map

Review the route on street, terrain, and FAA sectional-reference views with weather, current location, and FAA airspace overlays.

Airspace Profile

Project the route against terrain and FAA airspace to understand altitude constraints and controlled-airspace structure along the flight.

Airport Briefs

Load current FAA d-TPP airport diagram PDFs for route airports without storing chart files in the repository.

Aircraft Profiles

Use editable aircraft performance profiles so cruise, climb, and planning assumptions reflect the selected aircraft configuration.

Technical Overview

How the system is organized

High-Level Flow

Static Pages
index, checkpoints, map, aircraft
→
Browser State
drafts, checkpoints, nav-log snapshot
→
Local API
weather, aircraft, checkpoints, airspace, elevation
→
External Data
AviationWeather, FAA, USGS, local datasets

Frontend

public/js/app.js orchestrates the main nav-log workflow.

public/js/checkpoints.js manages checkpoint review and editing.

public/js/map.js handles route plotting, weather overlays, and export behavior.

Shared State

public/js/flightStore.js keeps flight drafts, checkpoints, nav-log state, and shared caches synchronized across page transitions.

Backend

index.js exposes local endpoints for weather, checkpoint generation, airspace lookup, elevation sampling, chart metadata, and aircraft profile CRUD.

Platform

Current stack

Application

  • Frontend: Static HTML/CSS/JS
  • Backend: Node.js + Express
  • Testing: Jest + Playwright

Core inputs

  • Weather: AviationWeather.gov
  • Variation: offline WMM via magvar
  • Terrain: USGS elevation sampling
  • Airspace: FAA airspace geometry and shelf values

Testing

Local-first change safety

Unit Tests

Jest covers weather normalization, fallback behavior, magnetic variation, and supporting services.

End-to-End Tests

Playwright validates route generation, planner behavior, route map behavior, airspace workflows, and shared state across pages.
